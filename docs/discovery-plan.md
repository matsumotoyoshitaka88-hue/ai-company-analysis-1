# Discovery Plan: 企業AI診断サービス

**作成日**: 2026-03-20
**プロダクトステージ**: 新規プロダクト
**目的**: リード獲得（別サービスへの導線）
**ディスカバリーの問い**: AIを使った企業診断サービスは、管理職に価値を提供し、本業サービスへのリード獲得につながるか？

---

## 1. ディスカバリーの背景

企業の管理職が自社の置かれている状況（財務・強み・IT等）を十分に把握できていない課題に対し、PEファンドが実施するDD（デューデリジェンス）のような診断をAIで民主化する無料サービスを構想。診断を入口として本業サービスへの商談創出を狙う。

## 5. 仮説の優先順位（Impact × Uncertainty）

### Leap of Faith（最優先検証）

| 優先度 | 仮説 | 理由 |
|--------|------|------|
| 1 | **F3** LLMでDD品質に近い分析を自動生成できる | サービスの根幹。不成立なら全アイデアが崩壊 |
| 2 | **V2** 既存の無料ツール/情報では不十分と感じている | わざわざ使う動機の有無 |
| 3 | **F1** 公開データだけで意味のある基礎診断が出せる | 手軽さの実現性 |
| 4 | **B1'** 診断利用者が本業サービスの商談につながる | リード獲得の前提 |
| 5 | **G2** 管理職がAI診断を信頼するか | 無料でも信頼なければ使われない |
| 6 | **B2'** 診断→本業サービスへの自然な導線が設計できる | CVRの前提 |

### Important（次フェーズで検証）

| 仮説 | 理由 |
|------|------|
| V1 管理職が課題と感じている | 一般的にYesだが「行動するほど」かは別 |
| F2 業界特化モデルの学習データ入手可能性 | 業界によって差がある |
| F4 競合データの合法的収集可能性 | 非上場企業だと困難 |
| G1 管理職へのリーチチャネル | BtoBのCACは高い傾向 |

## 6. 検証実験

### 実験1: プロトタイプ診断テスト

**検証仮説**: F3（LLMでDD品質の分析が可能か）、F1（公開データだけで価値ある診断が出せるか）

| 項目 | 内容 |
|------|------|
| 方法 | 実在の上場企業3-5社を対象に、有報・IR資料・ニュースからLLM（Claude）で診断レポートを手動生成。PEファンドDD経験者にレビューを依頼 |
| 成功基準 | DD経験者が「基礎診断として実用レベル」と評価（5段階で3以上） |
| 工数 | 1週間 |
| ツール | Claude API + 公開データ手動収集 |

### 実験2: 課題インタビュー

**検証仮説**: V2（既存ツールへの不満）、G2（AI診断への信頼感）

| 項目 | 内容 |
|------|------|
| 方法 | 管理職5-8名にオンラインインタビュー（30分）。「自社の状況把握方法」「不満」「AI診断への印象」を聴取。実験1のサンプルレポートを見せて反応を取得 |
| 成功基準 | 5名中3名以上が「現状に不満」かつ「AI診断を試したい」と回答 |
| 工数 | 2週間（リクルーティング含む） |

### 実験3: フェイクドア + 導線テスト

**検証仮説**: B1'（商談へのつながり）、B2'（自然な導線設計）

| 項目 | 内容 |
|------|------|
| 方法 | LPを作成。「無料AI企業診断」→ 簡易診断レポート → レポート末尾に本業サービスへのCTAを設置。広告 or SNSで管理職層に配信 |
| 成功基準 | 診断完了者の5%以上がCTAクリック、1%以上が問い合わせ送信 |
| 工数 | 2-3週間 |
| コスト | LP制作 + 少額広告費（5-10万円） |

---

## 7. ディスカバリータイムライン

```
Week 1      : 実験1（プロトタイプ診断レポート生成・レビュー）
Week 2-3    : 実験2（管理職インタビュー）※実験1の成果物を活用
Week 3-5    : 実験3（LP・フェイクドア）※実験1,2の学びを反映
Week 6      : 結果分析 → Build / Kill 判断
```

---

## 8. 判断フレームワーク

### Build 条件（すべて満たす場合）
- 実験1: DD経験者がスコア3以上
- 実験2: 3名以上が「試したい」
- 実験3: CTAクリック率5%以上

### Kill 条件（いずれかに該当する場合）
- 実験1: LLMの出力品質がスコア2以下で改善余地なし
- 実験2: 管理職が現状に不満を感じていない
- 実験3: CTAクリック率1%未満

### Pivot 条件（一部成立・一部不成立の場合）
- 実験1成功 + 実験3失敗 → 導線設計を再考、ターゲットセグメントを変更
- 実験2成功 + 実験1失敗 → 診断の対象領域・深さを変更して再テスト
- 全実験で微妙な結果 → ターゲットを管理職→経営企画部門に変更して再検証
